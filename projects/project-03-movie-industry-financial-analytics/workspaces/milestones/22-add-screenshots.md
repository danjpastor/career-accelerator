## Detailed workflow

1. Select only views that help a reviewer understand the project quickly.
2. Set readable dimensions, filters, and zoom.
3. Remove irrelevant application chrome and private information.
4. Capture consistent images with descriptive filenames.
5. Add captions or alt text that explain what each image proves.
6. Reference the correct relative paths from the README.
7. Test readability in the rendered public repository.
