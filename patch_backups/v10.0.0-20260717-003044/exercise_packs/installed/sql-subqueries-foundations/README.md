# SQL Subqueries: Foundations to Advanced

A deliberately paced optional Exercise Pack for learners who understand basic SQL but find nested queries difficult to read or build.

## What makes this pack different

- Starts by running the inner calculation alone.
- Uses the **result-shape** mental model: one value, one-column list, or table.
- Introduces only one new idea at a time.
- Provides progressive hints without immediately replacing your work.
- Explains common mistakes and the reasoning behind every official solution.
- Connects subqueries to joins and CTEs instead of treating them as unrelated topics.
- Ends with a realistic capstone while keeping every dataset small enough to inspect manually.

## Recommended path

1. Read Lessons 1–2 and complete Exercises 1–3.
2. Read Lesson 3 and complete Exercises 4–5.
3. Read Lesson 4 and complete Exercise 6.
4. Read Lesson 5 and complete Exercises 7–8 and 10.
5. Read Lessons 6–7 and complete Exercises 9, 11, and 12.

The pack is optional. Work slowly, run partial queries often, and use the Notes box to complete these sentences:

- **The inner query returns...**
- **The outer query uses that result to...**

Estimated total time: approximately 6–7 hours, which can be split across several sessions.
