# SQL Joins: Match, Preserve, and Combine Rows

This optional Career Accelerator Exercise Pack teaches SQL joins from first principles. It is designed for learners who can write basic `SELECT` queries but still confuse which join keeps which rows, why joins sometimes create duplicates, or how to combine more than two tables safely.

## Install the pack

1. Open **Learning → Exercises**.
2. Open the pack-management menu.
3. Choose **Install Exercise Pack…** and select the `.exercise-pack.zip` file.
4. To test folder installation instead, extract the ZIP and choose **Install Folder…**.

The pack installs separately from the bundled Subqueries pack. Reinstalling a newer version with the same `pack_id` preserves saved SQL, notes, and completion progress.

## Course structure

- 7 polished lessons
- 12 checked SQL exercises
- 7 small CSV datasets
- Approximately 360 minutes
- Very Beginner through Advanced Foundation

## Mental model used throughout

For every join, answer four questions before writing SQL:

1. What does one row represent in each table?
2. Which columns describe the relationship?
3. Which table's rows must be preserved?
4. Can one row match zero, one, or many rows on the other side?

The pack covers `INNER JOIN`, `LEFT JOIN`, the meaning of `RIGHT JOIN`, `FULL OUTER JOIN`, `CROSS JOIN`, multi-table joins, self joins, unmatched rows, `NULL`, one-to-many relationships, and fanout-safe aggregation.
