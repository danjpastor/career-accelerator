# DuckDB Exercise Workspace

Open SQL Companion and choose the **DuckDB Exercises** tab.

## Exercise files

Each exercise provides buttons for:

- Instructions (`README.md`)
- Starter SQL
- Validation guide
- Exercise-specific dataset folder
- A saved submission copy

**Create / Open Submission** copies the starter into
`practice/duckdb/submissions/` and opens that copy in VS Code or the
operating system's default editor. The starter file remains unchanged.

## Saving progress

Choose Not Started, In Progress, or Completed and select
**Save Exercise Progress**.

Completing an exercise:

- records the submission path and notes
- updates a matching sprint task when one exists
- makes the exercise available as SQL concept evidence
- adds an SQL Practice item to Demonstrated Evidence

The app warns when a completed submission still matches the starter
template.
